В рабочей конфигурации должны быть удалены все строки, кроме описывающих цепочки макросов.
Файл можно редактировать, когда игра запущена. При сохранении файла через текстовый редактор, изменения будут подтянуты автоматически.
Первый попавшийся список с ID клавиш Win32API https://asawicki.info/nosense/doc/devices/keyboard/key_codes.html


Формат:
<ID клавиши из Win32API> <single/hold/auto> <r/l/c/50/50-150>...
Разделение действий пробелами обязательно. Разделение цепочек действий переносами строки обязательно.
Первые 2 значения отвечают за кнопку для запуска цепочки и тип повтора цепочки. Всё, что идёт дальше - действия цепочки.

single - выполнить цепочку 1 раз.
hold - выполнение цепочки будет продолжаться до отпускания клавиши.
auto - выполнение цепочки будет продолжаться до повторного нажатия клавиши.

c - Вывести дебаг сообщение в чат
r - Отправить пакет Use Item (ПКМ) с текущим направлением взгляда игрока.
l - Отправить пакет Swing Arm (ЛКМ).
Число - Задержка в миллисекундах.
Число-число - Задержка в мс. Время будет каждый раз рандомиться в диапазоне между этими числами.


Примеры:

90 hold l 50
- Пока зажата клавиша Z, отправлять ЛКМ каждые 50мс.

36 single r r r 200 r 50 l 50 r 50 r 50 r 50 r c
- При нажатии Home, 1 раз выполнить цепочку из всех следующих действий. В конце выводится дебаг сообщение.

88 auto r 50 r 50 l 50 r 50 l 50 r 2300-2500
- Нажатие X переключает постоянное выполнение цепочки из всех следующих действий. В конце каждого выполнения происходит случайная задержка в 2.3-3 сек.


Вид файла рабочего конфига:
90 hold l 50
36 single r r r 200 r 50 l 50 r 50 r 50 r 50 r c
88 auto r 50 r 50 l 50 r 50 l 50 r 2300-2500
